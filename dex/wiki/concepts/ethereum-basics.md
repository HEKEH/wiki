# Ethereum 基础

理解 DEX 开发的前提是理解以太坊本身。

## 区块链是什么？

一个**全球共享的数据库**，所有人都可以读取，但写入需要遵循规则：

- 所有人看到相同的数据（共识）
- 写入数据需要付费（gas）
- 写入后不可篡改（最终性）
- 没有单一管理者控制（去中心化）

可以把它想象成一个所有人都有一份拷贝的 Excel 表格，任何修改都要经过大家同意才能写入。

## Ethereum 是什么？

Ethereum（以太坊）是一个**可编程的区块链**。和比特币只能转账不同，以太坊可以运行程序（智能合约），这使它能支撑 DeFi、NFT、DEX 等应用。

以太坊上有两种"账户"：

| 类型 | 由谁控制 | 有代码吗 | 例子 |
| ---- | -------- | -------- | ---- |
| **外部账户（EOA）** | 私钥持有者 | 无 | 你的 MetaMask 钱包 |
| **合约账户** | 智能合约代码 | 有 | Uniswap 的交易合约 |

## 地址、公钥、私钥

这三个概念是理解钱包安全的基础：

```
私钥 → 公钥 → 地址
（256位随机数） （椭圆曲线推导） （公钥的后20字节）
```

- **私钥**：掌控资金的终极密码。谁拥有私钥，谁就能花里面的钱。永远不要分享给任何人。
- **公钥**：由私钥推导而来，可以公开，但一般不需要直接使用。
- **地址**：`0x` 开头的 42 位字符串，你的"账号"，别人给你转账就填这个。

> 类比：私钥 = 银行卡密码，地址 = 银行卡号。知道密码就能取钱，知道卡号只能存钱。

## 交易（Transaction）

交易是改变链上状态的唯一方式。常见操作：

| 操作 | 需要付费？ | 说明 |
| ---- | ---------- | ---- |
| 转账 ETH | 是 | 从一个地址转到另一个地址 |
| 调用合约写入方法 | 是 | 执行 swap、approve 等 |
| 部署合约 | 是 | 将代码上传到链上 |
| 读取数据 | 否 | 查余额、查价格等（不叫交易，叫 call） |

交易流程：构造 → 签名 → 广播 → 等待打包 → 上链确认

## 测试网 vs 主网

| 网络 | 资金 | 用途 |
| ---- | ---- | ---- |
| **主网（Mainnet）** | 真实 ETH，有实际价值 | 生产环境 |
| **测试网（Sepolia 等）** | 假 ETH，免费获取 | 开发测试 |

开发时永远先在测试网验证，确认无误后再部署主网。

## EVM 链与非 EVM 链

以太坊、Polygon、BSC、Arbitrum、Avalanche 等都运行以太坊虚拟机（EVM），合约和 SDK 可直接复用。Solana、Tron 等非 EVM 链架构完全不同，需要换技术栈。详见 [[concepts/token-standards]]。
