# 硬件钱包

专用物理设备，私钥生成和存储在设备芯片内，即使连接的电脑被黑也无法取出私钥。

## 为什么需要硬件钱包？

[[entities/metamask|MetaMask]] 是热钱包，安全前提是"电脑未被入侵"。如果电脑被黑客控制：

- 键盘记录器可获取解锁密码
- localStorage 中的加密私钥可被读取
- 交易内容可被篡改（盲签攻击）

硬件钱包把这个前提去掉了——**即使电脑全被黑，私钥也出不来。**

## 工作原理

```
你的电脑（可能被黑）              硬件钱包（独立安全设备）
      │                                │
      │  1. 发送交易请求 ──────────→   │
      │                                │  2. 设备屏幕显示交易详情
      │                                │     收款地址、金额、合约调用
      │                                │
      │                                │  3. 你在设备上按按钮确认
      │                                │     私钥在芯片内签名
      │                                │
      │  4. 返回签名结果 ←──────────   │
      │                                │
      │  5. 广播签名后的交易            │  私钥永远不离开设备
      ▼                                ▼
```

关键：**交易细节显示在硬件设备的屏幕上，由你亲眼确认。** 电脑上的内容可以造假，但硬件屏幕上的内容无法被篡改。

## 热钱包 vs 冷钱包对比

| 维度 | 热钱包（MetaMask） | 硬件钱包（Ledger/Trezor） |
| --- | --- | --- |
| 私钥存储位置 | 浏览器 localStorage | 专用安全芯片 |
| 是否联网 | 始终在联网设备上 | 私钥从不触网 |
| 电脑被黑的安全性 | 不安全 | **安全** |
| 使用便利性 | 高，一键操作 | 较低，需要插设备、按按钮确认 |
| 适合场景 | 日常小额交易 | 大额资产存储 |
| 价格 | 免费 | 约 $60–$200 |

## 主流硬件钱包

| 产品 | 特点 |
| --- | --- |
| **Ledger Nano S Plus / X** | 市场份额最大，支持 5000+ 代币，Nano X 有蓝牙 |
| **Trezor Model One / T** | 开源固件，Model T 有触摸屏 |
| **Keystone Pro** | 气隙签名（QR 码通信），完全不接线 |

## 硬件钱包与 DEX 开发

在 DEX 前端中，硬件钱包通过 MetaMask 桥接——用户在 MetaMask 中选择"连接硬件钱包"，代码层面无需修改：

```javascript
// 对开发者来说，代码完全一样
const provider = new BrowserProvider(window.ethereum);
const signer = await provider.getSigner();

// 区别只在用户体验：
// MetaMask 热钱包：弹窗输入密码 → 确认
// 硬件钱包：弹窗 → 在设备上按按钮确认
```

## 安全注意事项

| 注意事项 | 说明 |
| --- | --- |
| **只从官方渠道购买** | 二手/第三方渠道的设备可能被预装恶意固件 |
| **验证设备屏幕内容** | 永远在设备屏幕上核实收款地址和金额，不要只看电脑屏幕 |
| **备份助记词** | 设备丢失 = 私钥丢失，必须手写助记词备份 |
| **固件更新** | 保持固件最新，修复安全漏洞 |

## 一句话

**热钱包管日常，硬件钱包管存钱。** 这是 DeFi 用户最基本的安全实践。
