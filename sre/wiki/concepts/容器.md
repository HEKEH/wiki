# 容器（Container）

容器是一种**便携式、轻量级的操作系统级虚拟化技术**。它通过 namespace 隔离不同软件的运行环境，并用镜像自包含运行环境，从而能很方便地在任何地方运行。

## 核心机制

- **隔离**：使用 Linux **namespace** 隔离不同软件的运行环境（与 [[entities/Namespace]] 的 K8s 资源隔离概念同名但不同层）。
- **自包含**：通过**镜像**打包应用及其运行环境，应用不依赖外部基础架构，解决"从开发到生产环境一致性"问题。
- **一对一**：体积小、启动快，通常每个镜像打包一个应用。
- **透明**：容器进程的生命周期由基础设施管理（而非被进程管理器隐藏在内部），便于监测和管理。管理容器部署 ≈ 管理应用部署。

## 主要优点（摘要）

敏捷创建部署、持续集成与快速回滚（镜像不可变）、开发与运维关注分离、环境一致性、可观测（应用级指标）、跨云跨 OS 可移植、以应用为中心、松耦合微服务、资源隔离与高利用率。

## 在 Kubernetes 中

容器不被单独调度，而是被组织进 [[entities/Pod]]；运行容器需要 Node 上的 container runtime（通过 CRI 接口，见 [[concepts/插件机制与可扩展性]]）。容器编排的整体思想见 [[concepts/容器编排]]。
