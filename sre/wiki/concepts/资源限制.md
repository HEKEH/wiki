# 资源限制

Kubernetes 通过 **cgroups** 提供容器资源管理，可限制每个容器的 CPU 和内存使用，实现可预测的应用性能与资源隔离。

## 设置方式

命令行（针对已有 [[entities/Deployment]]）：

```bash
kubectl set resources deployment nginx-app -c=nginx --limits=cpu=500m,memory=128Mi
```

等价于在 Pod 的容器 `spec` 中设置 `resources.limits`：

```yaml
spec:
  containers:
    - image: nginx
      name: nginx
      resources:
        limits:
          cpu: "500m"      # 0.5 个 CPU 核
          memory: "128Mi"
```

## 单位说明

- **CPU**：`500m` 表示 0.5 核（`1000m` = 1 核）。例中限制最多用 50% 的 CPU。
- **内存**：`128Mi` 表示 128 MiB（mebibyte）。

内存单位两套都支持，区别在进制——**带 `i` 是 2 的幂、不带是 10 的幂**：

| 二进制（2 的幂） | 十进制（10 的幂） |
| --- | --- |
| `Ki` / `Mi` / `Gi` / `Ti` | `k` / `M` / `G` / `T`（kilo 是小写 `k`） |

例如 `1Mi = 2²⁰ = 1,048,576` 字节，`1M = 10⁶ = 1,000,000` 字节（`Mi` 比 `M` 大约 4.9%）。所以 `128Mi ≈ 134 MB`。内存通常用 `Mi`/`Gi`（内存本按 2 的幂分配，更自然）。

> 底层由 cgroups 强制执行。资源限制是 [[entities/Node]] 上调度与资源隔离的基础，也体现了源文档所说"资源隔离：可预测的应用程序性能"（见 [[concepts/容器]]）。
