## 内存基础回顾

内存物理上是一大片能存 0/1 的存储单元网格，每个单元由一个二进制**地址**定位（"地址"一词源自硬件
设计——每个单元在矩阵里有唯一位置）。单比特只能表示两态，要表示更复杂的数据需更多位，所以计算机
不把地址关联到 1 个比特而是若干个：这个单位传统上叫**字(word)**，更常听到的叫**字节(byte)**，通常
8 位、可表示 256 种状态。对开发者而言，可以把内存看成**一个巨大的字节数组**，每个字节有唯一地址。

内存之所以处于现代计算机的核心，是因为**主存和 CPU 各核心内的寄存器，是 CPU 唯一能直接访问的通用
存储**。有 `load`/`store` 这类**以内存地址为参数**的机器指令让 CPU 读写指定内存位置；但**没有**任何
指令吃磁盘地址。因此正在执行的指令及其操作的数据都必须待在这类可直接访问的存储里——不在内存就得先
搬进来。哪怕"自增一个计数器"这种小操作也涉及多次访存：取指本身是一次读，`load` 是一次读，`store`
是一次写。

到此都是旧知识。今天的新问题还是老朋友——**并发让一切复杂起来**。

## 要确立的规则：地址空间神圣不可侵犯

并发不只影响 CPU 时间，也影响内存。程序在磁盘上，启动时必须载入内存，CPU 才能取指执行，于是成为
**进程**。进程还要内存放变量、临时值等；分配给它的全部内存称为它的**地址空间**。

**要确立的规则：默认情况下，一个进程永远不应能访问另一个进程的地址空间。** 为什么？设想两个并发
进程：第一个显示支付表单，用户输入存在该进程地址空间里的某个缓冲区。若不强制内存访问限制：

- 第二个（或任何）进程可用 `load`/`store` 写一小段代码，**读走**第一个进程的支付数据，拷进自己
  空间，经网络发给恶意方。考虑到我们常随手安装不知何人开发的应用，这后果很严重。
- 不只读危险：能访问就能**写**——恶意进程可改缓冲区，悄悄篡改支付金额或收款人。
- 更糟的是，无内存保护时一个进程甚至能改另一个进程、乃至**操作系统**的代码，注入恶意可执行代码，
  使其几乎为所欲为。

而且操作系统自身也靠内存来管理系统，所以**无保护的内存访问是任何现代 OS 都不能忽视的**。进程的
地址空间是神圣的：一个进程绝不能读或写另一个进程的内存。

## 为何不能纯软件实现

强制这条规则有多种途径，但**纯软件不行**。内存操作在进程运行时无时无刻不在发生；若 OS 要为当前用
CPU 的每个进程**手动检查每一次非法访问**，就得在用户进程的每次内存操作上跑额外代码——而指令本身又
要从内存取，于是用户每执行一条指令，OS 可能要多跑好几条来校验访存，带来**巨大性能开销**。所以**需要
硬件支持**。

## 硬件方案：基址与界限寄存器

不必改内存本身——进程运行时，内存单元只通过一组并行导线（**地址总线**）看到一串来自 CPU 的地址，
并不知道它们如何产生、作何用途。所以保护逻辑不放在内存里。用两个特殊寄存器来强制内存边界：

- **基址寄存器(base register)**：指向该进程**最小的合法物理地址**。
- **界限寄存器(limit register)**：定义该进程可寻址范围的**大小**。

判定地址是否落在自己的地址空间内，即检查：地址 **≥ 基址** 且 **< 基址 + 界限**。这与 OS 用软件
检查时的逻辑相同，只是这里翻译成了硬件。注意基址+界限是地址空间的**上界**，但该上界是**排他的**，
所以检查"小于"而非"小于等于"。

- 上界用一个**二进制加法器**把基址与界限相加即得。
- 我们要在**每一次**内存操作上校验地址；既然每个地址都经过地址总线，就可以在那里**拦截**地址。
- 用**二进制比较器**（接收两个二进制数 A、B，输出 A=B / A<B / A>B 三者之一）比较：一个比较器查
  地址 ≥ 基址，另一个查地址 < 上界。两个条件都为真地址才合法，故用一个 **AND 门**合并两路输出。

末端那根导线只在地址落入基址/界限定义的地址空间内时才有效。

## 一根导线如何拦住非法访问

回顾内存工作方式：把地址送上地址总线本身**不**触发内存操作；要真正读写必须激活 **read enable** 或
**write enable** 信号——读使能时内存把该地址内容放上数据总线，写使能时用数据总线上的数据覆写该
地址。于是最简单的拦截法，就是**用地址校验电路的输出去门控读/写信号**：若某指令试图访问进程地址
空间之外的地址，硬件会**阻止读/写使能信号抵达内存模块**。换言之，只有地址在当前进程地址空间界内，
内存才会收到读/写使能。

还有个好主意：加几个额外逻辑门**检测**进程试图访问非法地址的情形，这样电路不仅阻断该操作，还向 CPU
**触发一个硬件信号**，立即中断运行中的进程并唤起操作系统；OS 随即处理该违例，多半是**终止进程**。在
Unix 类系统上，当这个信号被 OS 捕获时，它终止进程并显示错误信息 **segmentation fault（段错误）**，
意为进程试图访问它无权访问的内存。

## OS 的唯一职责：设好寄存器

到这里应该很清楚：OS **不需要执行任何额外指令**就能让这套电路工作——因为接线方式如此，电路会自动
拦截任何内存操作（无论目的）并立刻校验。OS 的唯一职责，是在把 CPU 派给用户进程**之前，把正确的值
设进基址与界限寄存器**。这套电路当然都嵌在 CPU 内部。

要让一切如期工作，CPU 必须支持**运行模式**：基址、界限这类内存管理寄存器必须实现为**特权寄存器**，
只有操作系统能修改。于是这成了**上下文切换**时 OS 的又一职责：当 CPU 从一个进程重分配给另一个，OS
不仅捕获被中断进程的状态、恢复新进程的状态，**还要更新内存边界（基址/界限）以匹配新进程的地址
空间**。一切就绪后，OS 把 CPU 切换到**用户态**让用户进程运行；而在用户态下 CPU 无法修改特权寄存器，
所以进程只能使用 OS 明确分配给它的内存。

## 收尾与预告

这就是操作系统在硬件协助下阻止用户进程互相访问内存的基本思路。但内存的故事还没完：当多个进程运行、
没有更多内存可分配给新进程时怎么办？这要靠**分页(paging)**——一种让计算机使用比物理内存更多内存的
内存管理技术，留作后续一集。
