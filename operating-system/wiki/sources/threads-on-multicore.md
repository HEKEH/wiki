## 引言

本文是线程系列的续集，专门讨论线程在**多核处理器**系统中的作用。

先复习一个前置概念——**并发(concurrency)**：当有多个进程但只有一个 CPU 时，操作系统能在
它们之间以极快的速度轮换 CPU 访问权，从而让人感觉所有进程在同时运行。

上一集讲过线程之所以有用的两个关键原因：

1. **避免短任务苦等长任务**——同一程序里若一个任务耗时很久、另一个只要几毫秒，并发能保证
   短任务不必排在长任务后面干等。
2. **填满 IO 空隙**——任务等待 IO(文件读取、网络响应)时用不了 CPU；与其让 CPU 空转，操作
   系统可把这段时间分给另一个就绪任务。

所以**线程就是一种告知操作系统"同一进程内的多个任务可以并发运行"的方式**。例如邮件客户端
要同时：显示界面、监听键盘、上传附件、做语法检查、监测新邮件——为了好的体验，这些任务彼此
都不应该等待。

## 并发的极限与三种解法

实现并发是有代价的：如果并发任务数增长得太多，即便 CPU 轮换得再快，系统也会变得不流畅——
任务太多时，每个任务**重新获得 CPU 的间隔会变得太长**。有三种可能的解法：

1. **让 CPU 更快**——同样时间内处理更多工作，任务就能更快重获 CPU。但这并不完美：任务数继续
   增长又会回到原点；而且过去十年里**让 CPU 变快已越来越难**。
2. **更聪明地调度 CPU 访问**——这是更复杂的方案，值得单独一集来讲。
3. **加更多处理器**（与第一种一样属于"蛮力"思路）——单个处理器再快也扛不住太多任务时，那就
   直接增加处理器数量。

## 多处理器与术语

增加处理器有几种方式：

- 在同一主板上增加 **CPU 插槽**，容纳多个物理 CPU；
- 在单个封装/芯片内集成多个处理单元，即**多核处理器(multi-core)**；
- (较少见)把多个多核芯片装在同一主板上，两者结合。

术语上有些含糊：**CPU** 常指整个封装/芯片；而封装内的**每个核心(core)** 都是一个独立的
处理单元——本质上就是一个与其他核心**共享缓存等部件**的 CPU。无论如何，**每个核心在操作
系统看来都是一个独立的处理器**。

## 并发 vs 并行

考虑一个有 8 个线程的应用：

- 在**单核**系统上，并发只意味着这些线程的执行在时间上**交错(interleaved)**——因为处理核心
  一次只能执行一个线程。
- 在**多核**系统上，并发有了新含义：一些线程可以**真正同时运行**，因为系统能把每个线程分派到
  一个**独立的核心**上。这时我们处理的就不只是并发，而是**并行(parallelism)**。

注意两者的区别：

- **并发系统**支持多个任务，让所有任务都能取得**一些进展**。
- **并行系统**能让多个任务**真正同时**进行。

因此**可以有并发而没有并行**(单核就是如此)。并行系统的一大优势在于：多任务的流畅度不再
那么依赖"快速交错"制造的错觉。

## 为何线程成为基本执行单位

到这里就更能理解：为什么几乎所有现代操作系统都把**线程**(而非进程)当作最基本的执行单位。
多核已成标准，同一进程内的线程可以充分利用并行。

对程序员而言，这意味着：要让任务并行，只需把这些任务**声明为用线程并发**，剩下的交给操作
系统——没有空闲核心时就交错执行线程，有多核时就给每个线程分一个核心。这让程序更**可移植**：
不必为特定核心数编译。

但始终要记住两件事：

1. **核心数是固定的**。创建一千个线程并不意味着一千个任务真的并行运行。若系统有 n 个核心，
   任意时刻**最多有 n 个线程并行执行**。
2. 注意是"**最多** n"——因为线程要**竞争资源**。即使创建的线程数正好等于核心数，**其他进程的
   线程**也需要 CPU 时间；操作系统的目标之一是公平分配 CPU，所以它会限制我们自己有多少线程
   能真正并行。

## 性能：并行的另一个理由

这点很直白：若能真正同时运行多个任务，相比在单核上顺序执行，可以**显著减少完成这些任务的
总时间**。

## 两类并行

一般有两类并行：**数据并行**与**任务并行**。

- **数据并行(data parallelism)**：把**同一份数据**的子集分布到多个核心，每个核心对自己的子集
  执行**相同的操作**。
- **任务并行(task parallelism)**：分布的不是数据而是**任务/线程**——每个线程执行**不同的操作**。
  此时又有多种情形：不同线程可能操作**同一份数据**，也可能操作**不同数据**。

### 例子一：数据并行（找质数）

给定一个存有大量数字的数组，任务是找出其中所有质数。这需要遍历整个数组、逐个判断是否为质数。
关键在于：**判断一个数是否为质数的结果，不依赖于数组中其他数字的结果**——每个数都能独立处理。

有四核处理器时，就可把数据**均分成四份**、各分配给一个核心。这就是数据并行：所有核心对分布的
数据子集执行相同的操作。

> 注意：把工作分到四个核心**不一定**带来四倍性能。(作者把该例跑了一千万次，给出了每个子集的
> 平均计算耗时表；但"并行操作究竟能带来多少性能提升"被明确归为超出本视频范围。)

### 例子二：任务并行（同一数组上的多种统计）

同样的数据集，但这次要：找最小值、找最大值、计算算术平均、判断是否包含数字 101。这些操作**各自
都需要访问整个数据集**，所以把数据切成子集没有意义。可行的做法是：把**每个操作分配给一个核心**。
这就是任务并行——不同线程在**同一份数据**上执行**不同操作**。

同样，用四个核心不代表快四倍，但相比单核上**逐个顺序**执行这四种操作仍是显著提升：四核可同时
执行，缩短总时间。

## 小结

- 多核让并发升级为**并行**——线程可被分派到独立核心上真正同时运行。
- 并发≠并行：并发让所有任务都有进展，并行让任务真正同时；单核只有并发。
- 多核是线程成为现代基本执行单位的根本原因；程序员只需声明并发，可移植性由 OS 负责。
- n 核 → **最多** n 个线程并行（还要与其他进程竞争 CPU）。
- 并行分**数据并行**(同操作、分数据)与**任务并行**(分操作、同/异数据)；核心数翻倍不等于性能翻倍。
