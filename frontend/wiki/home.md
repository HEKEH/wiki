## Current State

Knowledge base initialized with a focus on **package registry experience design**, drawn from npmx.dev's innovation in the npm ecosystem. Key insight: competitive open-source alternatives can catalyze long-dormant platforms into shipping (npmjs.com shipped dark mode after 5 years of top-voted requests).

## Core Insights

1. **Supply chain transparency is a UX problem** — Registries should proactively surface install scripts, transitive sizes, and vulnerability trees, not leave it to external tools.
2. **Developer decision speed matters** — Resolved version ranges, format badges, and alternative suggestions reduce round-trips to the CLI.
3. **Decentralized social layers solve moderation** — AT Protocol integration lets registries add reviews/comments without running a moderation silo.
4. **URL compatibility lowers switching cost** — npmx's hostname-swap strategy (same as Invidious/Nitter) means zero migration friction for extensions and muscle memory.
